# Короткие решения для госов

Этот файл нужен для подготовки и объяснения.  
Полные файлы с кодом лежат рядом, но читать лучше отсюда.

## Java

1. Произведение справа  
Идём по массиву справа налево. В переменной храним произведение всех чисел справа. Для последнего элемента справа ничего нет, поэтому там 1.

2. Медиана общих элементов  
Так как оба массива отсортированы, используем два указателя. Если элементы равны — добавляем в список общих. Потом берём середину списка.

3. Сжатие диапазонов  
Сначала убираем повторы и сортируем числа. Потом идём подряд и собираем последовательности. Если подряд 3 и больше числа — пишем `start-end`, иначе пишем числа отдельно.

4. Слияние k списков  
Берём минимальный текущий элемент через очередь с приоритетом. Чтобы удалить дубликаты, запоминаем последнее добавленное число.

5. Числа хотя бы в двух массивах  
Считаем, сколько раз каждое число встретилось во внутренних массивах. Если счётчик >= 2, добавляем в ответ.

6. Гласные и цифры  
Одним циклом считаем гласные и цифры. Если цифр нет — ответ 0, иначе `vowels // digits`.

7. Удаление дублей максимум два раза  
Идём по отсортированному массиву и перезаписываем его. Новое число можно записать, если таких чисел до него было меньше двух.

8. Ransom note  
Считаем буквы в `magazine`, но не больше двух для каждой буквы. Потом тратим буквы на `ransomNote`.

9. Знак суммы квадратов  
Сумма квадратов не бывает отрицательной. Если есть хотя бы одно ненулевое число — ответ 1, иначе 0.

10. Запросы присваивания  
Для каждого запроса просто идём от `l` до `r` с шагом `k` и ставим значение `v`. Потом считаем сумму массива.

## Python

1. `DataKeeper`  
Храним исходный список и список уникальных значений в порядке первого появления.

2. Lucas  
Генератор считает числа последовательно. Рекурсия считает по формуле `L(n)=L(n-1)+L(n-2)`. Мемоизация ускоряет рекурсию.

3. `TableMapper`  
Первая строка — заголовки. Остальные строки превращаем в словари. Плохие строки пропускаем.

4. Временные ряды  
Из даты получаем год, месяц и день. Для каждой группы считаем среднее из предыдущего, текущего и следующего значения.

5. Декоратор  
Оборачивает функцию, печатает имя, аргументы, результат и время выполнения.

6. Фигуры  
У каждой фигуры свой метод `get_area()`. Общая функция просто складывает площади.

7. Плоский словарь  
Рекурсивно проходим словари и списки. Ключ собирается через точку.

8. Контекстный менеджер  
В `__enter__` открываем файл, в `__exit__` закрываем и печатаем количество байт.

9. Валидация  
Проверяем, что `values` — непустой список целых чисел, а `mode` допустимый. Потом считаем результат.

10. Матрица  
Строка берётся по индексу, столбец собирается циклом. Максимум, минимум и транспонирование делаются двойным циклом.

## Алгоритмы

1. Ближайшая большая высота  
Монотонный стек. Когда нашли высоту больше, закрываем все меньшие элементы из стека. Сложность `O(n)`.

2. Самый частотный символ  
Считаем буквы в словаре без `Counter`. Потом находим максимальную частоту и складываем все такие частоты. `O(n)`.

3. k-й элемент двух массивов  
Не сливаем массивы. Каждый раз отбрасываем часть одного массива. Сложность `O(log k)`.

4. Сортировка слиянием  
Делим список пополам, сортируем части, потом сливаем два отсортированных списка. `O(n log n)`.

5. Бинарный поиск и корень  
В массиве ищем первое вхождение. Для корня делим отрезок пополам, пока длина отрезка больше `eps`.

6. Теги  
Открывающие теги кладём в стек. Закрывающий тег должен совпасть с последним открытым.

## SQL

Главная идея:  
`JOIN` соединяет таблицы, `GROUP BY` группирует, `COUNT` считает, `ORDER BY` сортирует, `LIMIT` берёт первые строки.

Пример для топ-3 городов:

```sql
SELECT p.city, COUNT(*) AS books_count
FROM book b
JOIN publisher p ON p.id = b.id_publisher
GROUP BY p.city
ORDER BY books_count DESC
LIMIT 3;
```

## ML

Общий шаблон почти для всех задач:

1. Прочитать CSV через `pandas`.
2. Посмотреть первые строки и пропуски.
3. Выбрать признаки `X` и целевую переменную `y`.
4. Разделить на train/test.
5. Обучить модель.
6. Посчитать метрику и построить график.

Для классификации: `LogisticRegression`, `RandomForestClassifier`.  
Для регрессии: `LinearRegression`.  
Для текста: `TfidfVectorizer + LogisticRegression`.

## Анализ текстов

Для классификации текста берём `TF-IDF + LogisticRegression`.

Почему так:

- TF-IDF переводит текст в числа.
- Логистическая регрессия простая и понятная.
- Можно получить вероятность класса.

Для LLM-задач главное — вернуть строгий JSON, чтобы ответ можно было проверить автоматически.

## WEB

Не надо держать все 10 задач в одном огромном файле при объяснении.

Для каждой WEB-задачи структура одинаковая:

1. HTML — элементы на странице.
2. CSS — сетка, карточки, адаптив.
3. JS — обработчик клика / формы / localStorage.

Если спрашивают по задаче:

- Метрики: кнопка меняет числа через JS.
- Бронь: форма проверяется регулярками.
- Рецепты: ингредиенты складываются в объект-корзину.
- Тема: класс `dark` + сохранение в `localStorage`.
- Ремонт: цена пересчитывается при изменении чекбоксов.
- Видео: при клике открывается модальное окно.
- Покупки: массив товаров хранится в `localStorage`.
- Вкладки: кнопка меняет видимый блок.
- Лендинг: секции, hero, якорь.
- Никнейм: собираем набор символов и выбираем случайные.
